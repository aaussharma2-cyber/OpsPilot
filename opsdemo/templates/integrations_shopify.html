Security checklist
  • Domain validated as *.myshopify.com before any request is made; pasted admin URLs are normalized automatically
  • Access token encrypted at rest with Fernet, keyed from CREDENTIAL_ENCRYPTION_KEY when configured
  • Token never logged or included in error messages
  • API calls use HTTPS and Shopify Admin GraphQL {{ api_version }}
  • Rate limit (429) responses handled with automatic back-off and retry
  • Sync capped at 5,000 records per run to prevent runaway requests
  • All data sanitised before writing to local database
  • Note: Changing the encryption key will require re-saving Shopify credentials.
{# ── Sync actions ────────────────────────────────────────────────────────── #}

Sync customers → CRM

Pulls all Shopify customers and creates or updates records in the CRM (matched by email).

Sync orders → Sales & Invoices

Pulls all Shopify orders. Each order creates a Sale entry. Unpaid or pending orders also create an Invoice (prefixed SHP-).

Automate daily sync

To run syncs automatically, schedule the Flask CLI command. It runs without a browser session and uses the encrypted credentials saved above: 

flask --app app shopify-sync
