How it works

🔑
Personal Access Token

Create a classic PAT with the repo scope at github.com/settings/tokens. Your token is encrypted at rest — never stored in plain text.

📦
Create, then push

Click Create repository once to make the repo on GitHub. Then click Push to GitHub whenever you want to snapshot the current project files. Each push force-updates the branch.

🗂
What gets pushed

All project files respecting .gitignore. The SQLite database (instance/opsdemo.db) and secrets are excluded automatically.

⚠
Force push

Each push uses --force, which overwrites remote history. This is intentional for a single-developer snapshot workflow. If you need full git history, use the terminal directly.
